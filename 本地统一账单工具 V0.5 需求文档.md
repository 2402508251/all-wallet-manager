# 本地统一账单工具 V0.5 功能需求文档

## 一、项目定位

纯本地单机记账工具，**无云端、不上传任何用户隐私数据**；自动标准化归集微信个人、支付宝个人、建设银行信用卡流水，解决跨平台消费、白条 / 花呗信用还款、账户互转账务错乱问题。

## 二、功能需求总览

### FR-01 账单采集

#### FR-01-01 本地文件手动上传
1. 支持用户手动上传微信、支付宝、建设银行导出的官方账单文件
2. 支持的文件格式：`.xlsx`、`.csv`、`.xls`、`.zip`
3. 支持拖拽上传

#### FR-01-02 邮箱 IMAP 拉取账单
1. 支持用户配置邮箱 IMAP 授权码，手动点击【同步账单】按钮拉取
2. **无后台定时轮询**，仅用户主动触发并二次确认后执行
3. 仅过滤账单类邮件（发件人白名单 + 主题关键词 + 附件类型过滤），不遍历全部私人邮件
4. 仅处理账单附件（zip/xlsx/xls/csv/pdf等），忽略无关文件
5. **增量拉取机制**：记录上次成功拉取的时间戳或邮件 UID，后续仅拉取增量邮件，避免全量遍历
6. **实时进度反馈**：异步执行时前端需展示详细进度（如："正在连接..." -> "找到X封邮件" -> "下载附件(2/3)..." -> "解析完成"），防止用户误以为卡死
7. 用户可随时一键清除邮箱授权凭证
8. 提供常见邮箱配置模板（如：163、qq等），用户可一键配置，仅允许配置一个邮箱

#### FR-01-03 上传\拉取结果展示列表
1. 展示上传或拉取后的所有采集结果
2. 当文件类型为zip时显示输入解压密码框（在该列右侧）
3. 渠道自动检测：根据文件名关键词自动识别微信/支付宝/建行渠道
4. 检测失败时标记为"渠道未知"，用户可手动指定渠道
5. 用户手动点击解析后自动判断是否解压并解析
6. 解压后自动清理临时文件
7. 解析结果展示（成功条数、重复跳过条数、合并条数、待分配条数、未分类条数）右侧附带删除与重新解析按钮
8. 持久化存储解析记录：文件名、所属账户、解析结果、解析时间、账单唯一索引列表（用于删除该次解析结果账单与重新解析）

### FR-02 账单解析与标准化

#### FR-02-01 多渠道解析
1. 支持微信个人账单解析（xlsx/csv，含 AES 加密 ZIP 附件）
2. 支持支付宝个人账单解析（csv/xlsx，含 AES 加密 ZIP 附件）
3. 支持建设银行信用卡账单解析（.xls/pdf，含 AES 加密 ZIP 附件）

#### FR-02-02 统一标准化
1. 各渠道原始字段映射为统一账单模型
2. 时间格式标准化
3. 金额标准化（统一转为整数分存储）
4. 渠道原始枚举值映射为内部统一枚举

#### FR-02-03 去重与事务保护
1. 基于渠道 + 流水单号唯一索引自动去重，重复记录跳过
2. 整批导入使用数据库事务，异常时自动回滚
3. 每次导入生成唯一批次 ID，支持按批次回滚

### FR-04 账单归属分配

#### FR-04-01 自动分配
1. 默认有一个默认角色和默认家庭，所有未分配角色的账单自动分配给该角色，与默认家庭关联
2. 账单导入后，自动生成账户并分配给账户回填账户的账户名默认为账户标识与`` 账户标识（渠道+账户尾号/微信昵称+支付方式）
   银行卡	包含银行名+(尾号)	alipay-{尾号}-{银行卡名}_{银行卡尾号}
   余额	"余额"、"余额宝"	alipay-{尾号}-{支付方式}
   贷款类	"花呗"、"信用支付"等	alipay-{尾号}-{支付方式}
   其他优惠	立减、红包等非真实	alipay-{尾号}-优惠 (统一一个) ``

3. 一个账单只属于一个角色，一个角色可以关联多个家庭

#### FR-04-02 手动分配
1. 用户可手动选择/修改账户的所属角色与角色的所属家庭
2. 选择角色后，家庭下拉框自动过滤为该角色关联的家庭列表
3. 选择家庭后，角色下拉框自动过滤为该家庭下的角色列表
4. 支持批量选择多条账单统一分配角色

### FR-05 账务处理

#### FR-05-01 信用消费与还款追踪
1. 花呗/分付透支消费标记为信用消费类型
2. 还款扣款记录与信用账户自动关联（transfer_link_id）
3. 系统自动生成负债账户还款镜像记录
4. 报表支持一键隐藏内部还款转账，仅消费类型计入月度消费总额

#### FR-05-02 跨平台支付自动合并
1. 当支付宝/微信通过银行卡付款时，自动合并第三方支付与银行卡两条流水记录
2. **孤儿记录处理**：若仅导入第三方支付记录而未找到对应银行卡记录，标记为"待合并（孤立）"状态，等待后续导入银行卡账单时自动触发二次扫描配对；超期孤立记录支持用户手动确认按独立消费计入统计
3. 匹配成功后，保留银行卡记录，第三方支付记录逻辑删除（非物理删除，进入回收站）
4. 根据来自第三方的记录完善银行卡记录的商品说明追与记录交易对手信息
5. 支持合并回溯与撤销

#### FR-05-03 资产账户互转配对（分为角色内互转与家庭内互转）
1. 转出/转入记录自动配对（同一 transfer_link_id）
2. 转账手续费单独生成记录
3. **细化的跨渠道自动配对规则**：
   - **强匹配**：渠道原始备注/单号一致，自动配对。
   - **弱匹配**：金额相同 + 方向相反 + 时间差（即时渠道≤5分钟，跨行渠道≤1天） + 交易对方名称语义相似，标记为"待确认"由用户选择。
4. 多条弱匹配候选时严格禁止自动配对，交由人工判断

#### FR-05-04 建设银行信用卡特殊处理
1. 建行账单无全局唯一交易号，系统自动生成唯一标识

### FR-07 源账单与重新解析

#### FR-07-01 源账单存储
1. 导入时将每条原始记录完整字段数据以 JSON 形式存储
2. 与统一账单记录一一对应，支持溯源

#### FR-07-02 重新解析
1. 用户修改分类关键词、枚举映射等配置后，可基于源账单重新解析
2. 支持选择解析范围：全部/指定批次/指定渠道/指定时间范围
3. 重新解析前调用**全局快照引擎**创建备份，支持变更对比与一键回退
4. 用户可手动清理源账单数据释放存储空间

## 三、术语表
| 术语/缩写 | 全称/解释 |
| :--- | :--- |
| **trade_type** | 账务层交易类型（如：消费、退款、转账、还款、手续费），决定资金增减。 |
| **业务分类** | 统计层类别（如：餐饮美食、交通出行），决定消费统计饼图归集。 |
| **transfer_link_id** | 系统生成的转账关联号，用于将一笔转出的"支出"与另一笔转入的"收入"进行配对。 |
| **逻辑删除** | 不从数据库物理抹除数据，而是标记 `is_deleted=1`，在流水列表中隐藏，用于合并撤销等场景。 |
| **孤立记录** | 特指跨平台合并场景中，仅存在第三方支付记录，但未找到对应银行卡扣款记录的"单边账"。 |
| **全局快照引擎** | 统一管理数据变更前备份的模块，采用增量记录 Diff 的方式，防止频繁操作导致数据库体积暴增。 |
| **共享角色** | 为解决夫妻联名卡、家庭公用账户等场景，在家庭下建立的虚拟公共身份标识，用于挂载共享资金账户。 |

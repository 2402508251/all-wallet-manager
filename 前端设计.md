## 五、前端页面与路由设计

### 5.1 页面结构

```
┌──────────────────────────────────────────────────┐
│  顶部导航栏 (TopNav)                              │
│  [账单采集] [账单管理] [账务处理] [统计报表] [设置] │
├──────────────────────────────────────────────────┤
│                                                  │
│                  主内容区域                       │
│                  (router-view)                   │
│                                                  │
└──────────────────────────────────────────────────┘
```

> **归属分配已移除**：账单不可手动分配家庭/角色。role_id 从 account_id 自动获取（导入时缓存），
> 家庭通过 `account_id → accounts.role_id → roles.family_id` 动态关联。
> 账户归属在系统设置页的"账户管理"Tab 中管理。

**应用启动流程**：

```javascript
// main.js
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import { createRouter, createWebHashHistory } from 'vue-router'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'
import '@/styles/variables.css'
import '@/styles/reset.css'
import '@/styles/global.css'
import '@/styles/element-overrides.css'
import App from './App.vue'
import { ready } from '@/utils/bridge'
import routes from './router'

const router = createRouter({ history: createWebHashHistory(), routes })
const pinia = createPinia()

// 等待 PyWebView Bridge 就绪后再挂载 Vue 应用
ready().then(() => {
  const app = createApp(App)
  app.use(pinia)
  app.use(router)
  app.use(ElementPlus, { locale: zhCn })
  app.mount('#app')
})
```

> **说明**：使用 `createWebHashHistory` 而非 `createWebHistory`，因为 PyWebView 加载本地静态文件，
> 不支持 HTML5 History 模式的服务端回退。

### 5.2 路由表

| 路由路径 | 页面组件 | meta.title | meta.icon | 说明 |
|:---|:---|:---|:---|:---|
| `/` | CollectionView | 账单采集 | Upload | 默认首页，重定向至 `/collection` |
| `/collection` | CollectionView | 账单采集 | Upload | 文件上传 + 邮箱拉取 + 采集结果管理 |
| `/bills` | BillListView | 账单管理 | Document | 多条件筛选 + 详情抽屉 + 批量操作 |
| `/accounting` | AccountingView | 账务处理 | Connection | 信用消费 / 跨平台合并 / 转账配对 |
| `/reports` | ReportView | 统计报表 | DataAnalysis | 月度摘要 + 多维度图表 + 分类排行 |
| `/settings` | SettingsView | 系统设置 | Setting | 家庭/角色/账户/分类/邮箱/关键词/数据管理 |

**路由配置实现**：

```javascript
// router/index.js
const routes = [
  { path: '/', redirect: '/collection' },
  { path: '/collection', component: () => import('@/views/CollectionView.vue'), meta: { title: '账单采集', icon: 'Upload' } },
  { path: '/bills', component: () => import('@/views/BillListView.vue'), meta: { title: '账单管理', icon: 'Document' } },
  { path: '/accounting', component: () => import('@/views/AccountingView.vue'), meta: { title: '账务处理', icon: 'Connection' } },
  { path: '/reports', component: () => import('@/views/ReportView.vue'), meta: { title: '统计报表', icon: 'DataAnalysis' } },
  { path: '/settings', component: () => import('@/views/SettingsView.vue'), meta: { title: '系统设置', icon: 'Setting' } },
]
```

**导航守卫**：

| 守卫类型 | 场景 | 处理逻辑 |
|:---|:---|:---|
| `beforeEach` | 全局前置 | 更新 `document.title` 为 `meta.title + ' - 统一账单'` |
| `beforeEach` | Bridge 未就绪 | 若 `!bridge.isReady()` 则等待 `bridge.ready()` 后再继续导航 |
| `beforeLeave` | 页面有未保存表单 | `ElMessageBox.confirm('有未保存的更改，确认离开？')`，取消则 `next(false)` |

**交互流程**：

| 用户操作 | 触发 API | 前端行为 |
|:---|:---|:---|
| 拖拽/选择文件上传 | `upload_files` | 文件校验 → 上传 → 创建采集记录 → 刷新列表 |
| 点击【同步账单】 | `fetch_email_bills` | 二次确认弹窗 → 返回 task_id → ProgressPanel 展示进度 |
| 监听 `task_progress` | — | 更新 ProgressPanel 进度条 + 步骤文字 |
| 监听 `task_done` | — | ElNotification 通知 + 刷新采集列表 |
| 点击【解析】 | `parse_collection` | 返回 task_id → 监听进度 → 解析完成后展示 ParseResultDialog |
| 渠道未知 → 手动选择渠道 | `set_channel_manual` | 下拉选择渠道 → 更新记录渠道 → 刷新行 |
| ZIP 文件 → 输入密码 | `set_zip_password` | 弹窗输入密码 → 尝试解压 → 成功则刷新状态 |
| 建行账单 → 设置账户标识 | `set_account_tag` | 弹窗输入标识 → 创建/匹配账户 → 刷新状态 |
| 点击【邮箱配置】 | `get_email_configs` | 弹窗加载邮箱列表 → 新增/编辑/测试/删除 |
| 测试邮箱连接 | `test_email_connection` | 按钮loading → 返回成功/失败提示 |
| 删除邮箱配置 | `delete_email_config` | 二次确认 → 可选清除授权码 → 刷新列表 |

- 家庭/角色变更 → 所有图表和摘要卡片联动刷新

**交互流程**：

| 用户操作 | 触发 API | 前端行为 |
|:---|:---|:---|

| 悬停柱状图/折线图 | — | ECharts tooltip 显示该日/月收支明细金额 |
| 页面首次加载 | `get_monthly_summary` + `get_category_distribution` + `get_trend_data` | 默认当月 + 全部家庭 → 并行加载 |

### 5.6 Bridge 通信封装（bridge.js）

```javascript
// utils/bridge.js

// PyWebView API 是否就绪检测
export function isReady() {
  return !!window.pywebview?.api
}

// 等待 PyWebView API 就绪（Vue app mount 前调用）
export function ready() {
  return new Promise((resolve) => {
    if (window.pywebview?.api) return resolve()
    window.addEventListener('pywebviewready', () => resolve())
  })
}

// 通用 API 调用封装（统一错误处理 + 返回值解包）
export async function call(method, params = {}) {
  const api = window.pywebview?.api
  if (!api || !api[method]) {
    throw new Error(`API 方法不可用: ${method}`)
  }
  const result = await api[method](params)
  if (result.success === false) {
    throw new Error(result.message || '操作失败')
  }
  return result.data
}

// 事件监听封装
export function on(eventName, handler) {
  window.addEventListener(eventName, (event) => handler(event.detail))
}

export function off(eventName, handler) {
  window.removeEventListener(eventName, (event) => handler(event.detail))
}

// 便捷方法：监听任务进度
export function onTaskProgress(taskId, onProgress, onDone) {
  const progressHandler = (data) => {
    if (data.task_id === taskId) onProgress(data)
  }
  const doneHandler = (data) => {
    if (data.task_id === taskId) {
      off('task_progress', progressHandler)
      off('task_done', doneHandler)
      onDone(data)
    }
  }
  on('task_progress', progressHandler)
  on('task_done', doneHandler)
  return () => {
    off('task_progress', progressHandler)
    off('task_done', doneHandler)
  }
}
```

**使用示例**：

```javascript

  task_id,
  (data) => { progress.value = data },   // 更新进度
```

```javascript
// composables/useECharts.js
import { ref, onMounted, onBeforeUnmount, watch } from 'vue'
import { echarts } from '@/utils/echarts'

export function useECharts(chartRef, optionGetter) {
  let chart = null

  const initChart = () => {
    if (chartRef.value) {
      chart = echarts.init(chartRef.value)
      chart.setOption(optionGetter())
    }
  }

  const updateChart = () => {
    if (chart) chart.setOption(optionGetter())
  }

  const handleResize = () => chart?.resize()

  onMounted(() => {
    initChart()
    window.addEventListener('resize', handleResize)
  })

  onBeforeUnmount(() => {
    window.removeEventListener('resize', handleResize)
    chart?.dispose()
  })

  return { updateChart, handleResize }
}
```

#### 5.10.4 作用域约定

| 样式类型 | 作用方式 | 说明 |
